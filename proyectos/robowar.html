RoboWar 2025: Mini-Sumo

Primera competencia regional: Un reto de improvisación, integración de hardware y gestión de crisis.

01.

Contexto y Retrospectiva

Este proyecto corresponde a mi primera participación en una competencia regional de robótica. Aunque contaba con experiencia en KidWind Challenge, enfrenté por primera vez los retos específicos de un robot de sumo en condiciones reales. El robot funcionó, pero presentó limitaciones importantes en organización de cables y desempeño debido a errores de planificación.


La Falla Crítica: Debido a retrasos en la planificación y adquisición de componentes, no recibí a tiempo la PCB diseñada para integrar el Arduino Nano y el puente H TB6612FNG. Esto obligó a improvisar con un Arduino Nano Screw Terminal, transformando el diseño limpio planeado en un reto de gestión de cables y espacio bajo presión técnica.

03.

Integración CAD y Adaptación de Diseño

Debido a las limitaciones de tiempo, el diseño mecánico no se realizó desde cero. Se seleccionaron modelos 3D de alta calidad del repositorio GrabCAD para el chasis principal y la tapa superior.


El trabajo técnico: Se utilizaron estos modelos como base en Fusion 360 y se realizaron modificaciones críticas en las geometrías para alojar con precisión los soportes de los sensores Sharp y QTR, así como adaptar el montaje de los micromotores N20, asegurando la robustez estructural para el combate.

04.

Lógica de Control y Filtrado

En lugar de usar lecturas crudas, implementé un filtro de promedio móvil para suavizar las señales de los sensores Sharp, evitando ataques falsos causados por el ruido eléctrico o rebotes de luz en la arena.

logic_main.ino

        // Filtro y busqueda
        void loop() {
          // Suavizado de señal: (Valor_anterior * 3 + Valor_actual) / 4
          fIzq = (fIzq * 3 + analogRead(DIST_LEFT)) / 4;
          fDer = (fDer * 3 + analogRead(DIST_RIGHT)) / 4;
          fFront = (fFront * 3 + analogRead(DIST_FRONT)) / 4;
        
          // Prioridad 1, evasion de bordes
          if (sensorValues[0] < 1000 || sensorValues[1] < 1000) {
            retroceder(255); delay(500);
            return;
          }
        
          // Prioridad 2, ataque si hay deteccion delante del sensor
          if (fFront > limiteOponente) {
            avanzar(255);
            return;
          }
        }

        #define DIST_LEFT A1
        #define DIST_RIGHT A0
        #define DIST_FRONT A2
        
        #include 
        
        QTRSensors qtr;
        
        const uint8_t SensorCount = 2;
        uint16_t sensorValues[SensorCount];
        
        // pines del driver
        #define AIN1 3
        #define AIN2 4
        #define PWMA 5
        #define BIN1 8
        #define BIN2 7
        #define PWMB 6
        
        // variables
        int distIzq, distDer, distFront;
        int limiteOponente = 350;
        int limiteAtaque = 700;
        
        int ultimoLado = 1;
        unsigned long tiempoUltimoDetectado = 0;
        
        int persistencia = 400;
        
        void setup() {
          pinMode(AIN1, OUTPUT);
          pinMode(AIN2, OUTPUT);
          pinMode(BIN1, OUTPUT);
          pinMode(BIN2, OUTPUT);
          pinMode(PWMA, OUTPUT);
          pinMode(PWMB, OUTPUT);
        
          qtr.setTypeRC();
          qtr.setSensorPins((const uint8_t[]){2, 9}, SensorCount);
        
          Serial.begin(9600);
        }
        
        // mover al frente
        void avanzar(int v) {
          digitalWrite(AIN1, HIGH);
          digitalWrite(AIN2, LOW);
          digitalWrite(BIN1, HIGH);
          digitalWrite(BIN2, LOW);
          analogWrite(PWMA, v);
          analogWrite(PWMB, v);
        }
        
        // ir hacia atras
        void retroceder(int v) {
          digitalWrite(AIN1, LOW);
          digitalWrite(AIN2, HIGH);
          digitalWrite(BIN1, LOW);
          digitalWrite(BIN2, HIGH);
          analogWrite(PWMA, v);
          analogWrite(PWMB, v);
        }
        
        // girar izquierda
        void girarIzquierda(int v) {
          digitalWrite(AIN1, LOW);
          digitalWrite(AIN2, HIGH);
          digitalWrite(BIN1, HIGH);
          digitalWrite(BIN2, LOW);
          analogWrite(PWMA, v);
          analogWrite(PWMB, v);
        }
        
        // girar derecha
        void girarDerecha(int v) {
          digitalWrite(AIN1, HIGH);
          digitalWrite(AIN2, LOW);
          digitalWrite(BIN1, LOW);
          digitalWrite(BIN2, HIGH);
          analogWrite(PWMA, v);
          analogWrite(PWMB, v);
        }
        
        void loop() {
        
          // leer linea
          qtr.read(sensorValues);
          uint16_t izq = sensorValues[0];
          uint16_t der = sensorValues[1];
        
          bool bordeIzq = izq < 1000;
          bool bordeDer = der < 1000;
        
          // leer distancia
          distIzq = analogRead(DIST_LEFT);
          distDer = analogRead(DIST_RIGHT);
          distFront = analogRead(DIST_FRONT);
        
          unsigned long ahora = millis();
        
          // suavizar sensores
          static int fIzq = 0, fDer = 0, fFront = 0;
          fIzq = (fIzq * 3 + distIzq) / 4;
          fDer = (fDer * 3 + distDer) / 4;
          fFront = (fFront * 3 + distFront) / 4;
        
          // si ve borde se salva primero
          if (bordeIzq && !bordeDer) {
            retroceder(255);
            delay(500);
            girarDerecha(255);
            delay(150);
            return;
          }
        
          if (bordeDer && !bordeIzq) {
            retroceder(255);
            delay(500);
            girarIzquierda(255);
            delay(150);
            return;
          }
        
          if (bordeIzq && bordeDer) {
            retroceder(255);
            delay(500);
            return;
          }
        
          // si detecta enfrente ataca
          if (fFront > limiteOponente) {
            avanzar(255);
            tiempoUltimoDetectado = ahora;
            return;
          }
        
          // si lo ve a un lado gira
          if (fIzq > limiteOponente && fDer < limiteOponente - 30) {
            girarIzquierda(255);
            ultimoLado = -1;
            tiempoUltimoDetectado = ahora;
            return;
          }
        
          if (fDer > limiteOponente && fIzq < limiteOponente - 30) {
            girarDerecha(255);
            ultimoLado = 1;
            tiempoUltimoDetectado = ahora;
            return;
          }
        
          // si hay ruido sigue con lo ultimo
          if (fIzq > limiteOponente && fDer > limiteOponente) {
            if (ultimoLado == 1) girarDerecha(255);
            else girarIzquierda(255);
            tiempoUltimoDetectado = ahora;
            return;
          }
        
          // si no ve nada sigue buscando
          if (ahora - tiempoUltimoDetectado < persistencia) {
            if (ultimoLado == 1) girarDerecha(255);
            else girarIzquierda(255);
          } else {
            if ((ahora / 2000) % 2 == 0) girarDerecha(255);
            else girarIzquierda(255);
          }
        
          delay(10);
        }

06.

Lecciones Aprendidas

  • Gestión del Cableado: La improvisación con el Screw Terminal me enseñó la importancia crítica de diseñar pensando en la PCB y el ruteo eléctrico desde las etapas iniciales del CAD para evitar desorganización ("nidos de cables") que dificultan el mantenimiento.
  • Planificación de Tiempos de Entrega: Aprender a anticipar los tiempos de fabricación y envío de PCBs para evitar improvisaciones críticas de último momento que afectan el diseño final.
  • Filtrado de Sensores: La implementación del filtro promedio simple en el código demostró la necesidad de limpiar las lecturas analógicas de distancia Sharp para evitar falsos positivos de ataque en entornos ruidosos como la arena de competencia.
  • Integración Hardware-Software: Adaptar la lógica de combate sobre la marcha para un hardware que no era el planificado desarrolló habilidades de programación y depuración bajo presión técnica.
