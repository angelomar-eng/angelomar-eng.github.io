RoboWar 2025: Mini-Sumo

Primera competencia regional: Un reto de improvisación, integración de hardware y gestión de crisis.

01.

Contexto y Retrospectiva

Este proyecto corresponde a mi primera participación en una competencia regional de robótica. Aunque contaba con experiencia en KidWind Challenge, enfrenté por primera vez los retos específicos de un robot de sumo en condiciones reales. El robot funcionó, pero presentó limitaciones importantes en organización de cables y desempeño debido a errores de planificación.


La Falla Crítica: Debido a retrasos en la planificación y adquisición de componentes, no recibí a tiempo la PCB diseñada para integrar el Arduino Nano y el puente H TB6612FNG. Esto obligó a improvisar con un Arduino Nano Screw Terminal, transformando el diseño limpio planeado en un reto de gestión de cables y espacio bajo presión técnica.

03.

Integración CAD y Adaptación de Diseño

Debido a las limitaciones de tiempo, el diseño mecánico no se realizó desde cero. Se seleccionaron modelos 3D de alta calidad del repositorio GrabCAD para el chasis principal y la tapa superior.


El trabajo técnico: Se utilizaron estos modelos como base en Fusion 360 y se realizaron modificaciones críticas en las geometrías para alojar con precisión los soportes de los sensores Sharp y QTR, así como adaptar el montaje de los micromotores N20, asegurando la robustez estructural para el combate.

04.

Programación y Lógica de Sensores

Implementación de algoritmos de búsqueda y ataque, ruteando la señal de los Sharp infrarrojos y los QTR de contraste. Incluye una función de filtrado promedio para reducir el ruido de las lecturas analógicas de distancia.


// --- Sensores ---
#define DIST_LEFT  A1
#define DIST_RIGHT A0
#define DIST_FRONT A2

#include 
QTRSensors qtr;
uint16_t lineSensors[2];

// --- Pines motores (simplificado) ---
#define AIN1 3
#define AIN2 4
#define PWMA 5
#define BIN1 8
#define BIN2 7
#define PWMB 6

// --- Variables clave ---
int limite = 350;
int ultimoLado = 1; // 1 = derecha, -1 = izquierda
unsigned long ultimoDetectado = 0;
int persistencia = 400;

// --- Movimiento (simplificado) ---
void move(int izq, int der) {
  digitalWrite(AIN1, izq > 0); digitalWrite(AIN2, izq <= 0);
  digitalWrite(BIN1, der > 0); digitalWrite(BIN2, der <= 0);
  analogWrite(PWMA, abs(izq));
  analogWrite(PWMB, abs(der));
}

// --- Loop principal ---
void loop() {

  // Lectura línea (borde)
  qtr.read(lineSensors);
  bool bordeIzq = lineSensors[0] < 1000;
  bool bordeDer = lineSensors[1] < 1000;

  // Lectura distancia
  int rawL = analogRead(DIST_LEFT);
  int rawR = analogRead(DIST_RIGHT);
  int rawF = analogRead(DIST_FRONT);

  // --- Filtro (clave) ---
  static int fL=0, fR=0, fF=0;
  fL = (fL*3 + rawL)/4;
  fR = (fR*3 + rawR)/4;
  fF = (fF*3 + rawF)/4;

  unsigned long t = millis();

  // --- 1. EVASIÓN DE BORDE ---
  if (bordeIzq || bordeDer) {
    move(-255, -255);
    delay(400);

    if (bordeIzq) move(255, -255);
    else move(-255, 255);

    delay(150);
    return;
  }

  // --- 2. ATAQUE FRONTAL ---
  if (fF > limite) {
    move(255, 255);
    ultimoDetectado = t;
    return;
  }

  // --- 3. DETECCIÓN LATERAL ---
  if (fL > limite && fR < limite) {
    move(-200, 200);
    ultimoLado = -1;
    ultimoDetectado = t;
    return;
  }

  if (fR > limite && fL < limite) {
    move(200, -200);
    ultimoLado = 1;
    ultimoDetectado = t;
    return;
  }

  // --- 4. PERSISTENCIA (MEMORIA) ---
  if (t - ultimoDetectado < persistencia) {
    move(ultimoLado * 200, -ultimoLado * 200);
  } 
  else {
    // búsqueda automática
    if ((t/2000)%2 == 0) move(200, -200);
    else move(-200, 200);
  }

  delay(10);
}

06.

Lecciones Aprendidas

  • Gestión del Cableado: La improvisación con el Screw Terminal me enseñó la importancia crítica de diseñar pensando en la PCB y el ruteo eléctrico desde las etapas iniciales del CAD para evitar desorganización ("nidos de cables") que dificultan el mantenimiento.
  • Planificación de Tiempos de Entrega: Aprender a anticipar los tiempos de fabricación y envío de PCBs para evitar improvisaciones críticas de último momento que afectan el diseño final.
  • Filtrado de Sensores: La implementación del filtro promedio simple en el código demostró la necesidad de limpiar las lecturas analógicas de distancia Sharp para evitar falsos positivos de ataque en entornos ruidosos como la arena de competencia.
  • Integración Hardware-Software: Adaptar la lógica de combate sobre la marcha para un hardware que no era el planificado desarrolló habilidades de programación y depuración bajo presión técnica.
