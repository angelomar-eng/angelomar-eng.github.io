04.

Lógica de Control Progresivo


// Configuración de PWM y Control de Motores
void setup() {
    ledcSetup(0, 20000, 10); // 20kHz, 10-bit resolution
    ledcAttachPin(ENA, 0);
    ledcAttachPin(ENB, 1);
}

void loop() {
    if (myGamepad && myGamepad->isConnected()) {
        // Mapeo de aceleración progresiva R2 (Throttle)
        int r2map = constrain(map(myGamepad->throttle(), 0, 740, 0, 1023), 0, 1023);
        int l2map = constrain(map(myGamepad->brake(), 0, 900, 0, 1023), 0, 1023);

        if (myGamepad->dpad() == 4) driveMotor(2, 900); // Giro Der
        else if (r2map > 50) driveMotor(1, r2map);      // Adelante
        else if (l2map > 50) driveMotor(-1, l2map);     // Reversa
        else driveMotor(0, 0);
    }
}

La V2 utiliza una resolución de 10 bits para el PWM, permitiendo un control milimétrico del empuje. Se implementó una función driveMotor universal para gestionar las 5 direcciones posibles del robot.

  • Frecuencia: 20 kHz para evitar ruido auditivo.
  • Seguridad: vTaskDelay(1) para estabilidad del kernel RTOS del ESP32.

Manufactura Aditiva e Iteración

Documentación del proceso de impresión 3D del prototipo inicial.

Análisis de Falla Técnica: El prototipo inicial (Gris) falló estructuralmente durante las pruebas de impacto debido a un bajo porcentaje de Infill. La versión final de competencia se reimprimió en PLA Negro con un relleno del 60% y patrones giroides para maximizar la absorción de energía.
